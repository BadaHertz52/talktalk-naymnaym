# Issue Implement Plan (톡톡냠냠 전용)

당신은 **이슈 착수 전 계획 수립자**입니다. 코드를 직접 작성하지 않고,
(1) 이슈를 이해한 바를 짧게 요약 → (2) 토큰 효율 스케줄을 짜서
→ (3) `implement-review-loop` 스킬을 호출하는 것까지만 합니다.

---

## 0단계 — 입력 파싱

프롬프트에서 다음을 추출한다.

- **역할/목표**: "역할:", "목표:" 뒤 문장
- **이슈 번호**: "관련 이슈: #N" 또는 "이슈 #N"
- 이슈 번호가 있으면 `gh issue view {N} --json title,body,labels,milestone` 로
  본문을 가져온다. 실패하거나 이슈 번호가 없으면 프롬프트 내용만으로 진행한다
  (실패를 이유로 멈추지 않는다).

---

## 1단계 — 이해 요약 (3~5줄, 그 이상 쓰지 않는다)

아래 내용만 짧게 정리해 출력한다. 설명이 코드보다 길어지지 않게 한다.

- 무엇을 구현/연결해야 하는지 (핵심 한 줄)
- 어떤 디렉터리/트랙에 걸치는지 (`src/game/`, `src/pages/`, `src/hooks/` 등)
- **변경 통제 대상**(`src/types/`, `src/stores/`, `src/routes/`) 수정이 필요한지
  여부 — 필요하면 CLAUDE.md 규칙에 따라 **사용자 승인 + 전용 커밋/PR 분리**가
  선행되어야 함을 한 줄로 명시
- `src/game/` 관련 작업이면 "React import 없는 순수 TS" 제약을 한 줄로 상기

---

## 2단계 — 토큰 효율 스케줄 설계

[[delegate-to-sonnet-agents]] 원칙을 그대로 적용한다.

| 원칙 | 적용 |
|---|---|
| 구현 담당 | `Agent` tool, `subagent_type: weaver-frontend`, **`model: "sonnet"` 명시** |
| 소규모 파일 묶기 | 같은 통합 지점(예: eraser+progress, 여러 페이지의 훅 연결)에 걸린 소규모 파일은 팀을 나누지 않고 **한 에이전트 호출에 묶는다** — 루프당 고정 비용(컨텍스트 재구성) 절감 |
| XS 작업 | 타입 몇 줄 정리, 상수 추가처럼 초소형 변경은 서브에이전트 호출 없이 **메인 세션이 직접** 처리한다고 명시 |
| 변경 통제 파일 | `types/`, `stores/`, `routes/` 수정이 섞여 있으면 반드시 **별도 팀 + 별도 커밋**으로 분리하고, 사용자 승인 이후에만 해당 팀을 실행 |
| 반복 횟수 | 기본 1회. 변경 통제 파일이 걸려 있거나(승인→반영 2단계 필요) 이전 트랙 산출물과의 인터페이스 정합 확인이 필요하면 2회 제안 |

이 표를 바탕으로 **팀 목록**(팀명 · 담당 파일 · 이유)을 3~6줄 이내로 나열한다.
팀이 1개뿐이면 표 없이 한 줄로 끝낸다.

---

## 3단계 — implement-review-loop 호출

Skill tool로 `implement-review-loop`을 호출한다. 이때 사용자에게 다시 파라미터를
묻지 않고, 2단계에서 정한 값을 그대로 전달한다 (0단계 파라미터 수집을 건너뛰는 것과
동일한 효과). 호출 프롬프트에 포함할 내용:

```
작업: <1단계 이해 요약 핵심 한 줄> (이슈 #N)
구현 서브에이전트: weaver-frontend (model: sonnet)
팀 분해: <2단계 팀 목록>
리뷰 방식: review-orchestrator
반복 횟수: <N>회
변경 통제 파일 포함 여부: <있음/없음 — 있으면 사용자 승인 후 전용 커밋 안내>
```

호출 직전에 사용자에게 위 요약을 한 번 보여주고 바로 진행한다
("이견 있으면 말씀해 주세요" 한 줄만 덧붙이고, 응답을 기다리지 않고 루프를 시작한다 —
막을 이유가 있으면 사용자가 끊는다).

---

## 원칙

- 이 스킬 자신은 코드를 읽거나 수정하지 않는다.
- 이해 요약과 스케줄 설계를 합쳐 **10줄을 넘기지 않는다** — 넘으면 팀을 더 크게 묶어라.
- `implement-review-loop` 호출 이후의 진행(이터레이션, 리뷰 선택 등)은 그 스킬의 규칙을 그대로 따른다.
